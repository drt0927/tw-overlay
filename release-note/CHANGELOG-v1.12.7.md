# 📦 TW-Overlay Release Note (v1.12.7)

**배포일:** 2026.05.15

이번 버전에서는 기존 숙제 체크리스트(Contents Checker) 시스템을 **다중 캐릭터 지원**으로 전면 개편하고, 내부 데이터 구조를 최적화하여 안정성을 크게 향상시켰습니다.

## ✨ 새 기능 (Added)
- **다중 캐릭터 관리 지원:**
  - 숙제 체크리스트에 캐릭터 프리셋을 여러 개 등록하고 관리할 수 있는 기능 추가.
  - 모든 캐릭터의 숙제 현황을 한 화면에서 매트릭스 형태로 직관적으로 확인할 수 있는 'Matrix View' 뷰 모드 제공.
  - Shift + 마우스 휠 스크롤을 통한 가로 스크롤 이동 지원.
- **캐릭터별 숙제 진행도 필터링:**
  - 캐릭터별로 안 하는 숙제를 회색 배경(N/A)으로 처리하여 진행도(% 및 진행 바) 계산에서 제외하는 토글 기능 추가.
- **다이어리 로그 연동 고도화:**
  - 숙제 완료 시 모험일지에 기록되는 로그에 `[본캐] 머루` 와 같이 수행한 캐릭터 이름이 태그로 표시되도록 연동.

## 🛠 개선 사항 (Changed)
- **데이터 구조 SSOT(단일 진실 공급원) 일원화:**
  - `ContentsCheckerItem`의 완료 상태 저장 구조를 캐릭터별 상태를 관리하는 `completedState` 딕셔너리로 완전히 일원화. 
  - 과거 단일 캐릭터 시절의 `isCompleted`, `lastCompletedAt` 필드를 제거하고, 앱 실행 시 기존 데이터를 자동으로 본캐(MAIN_CHAR_ID)의 상태로 안전하게 마이그레이션하도록 로직 개선.
- **게임 종료 리마인더(Game Exit Reminder) 고도화:**
  - 게임 종료 안내 화면에서 숙제가 남은 캐릭터가 여러 명일 경우, 모든 캐릭터의 미완료 숙제 목록을 캐릭터 배지와 함께 한눈에 렌더링하도록 개선.
- **최초 사용성 강화:**
  - 새 버전 설치 시 기본 제공되는 캐릭터 프리셋(`char-main`: '본캐') 자동 생성.

## 🐛 버그 수정 (Fixed)
- 기타 내부 구조 개편 과정에서 식별된 TypeScript 컴파일 오류 및 타입 의존성(`MAIN_CHAR_ID`) 문제 해결.
