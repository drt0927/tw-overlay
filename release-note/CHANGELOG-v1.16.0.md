# TW-Overlay v1.16.0 릴리즈 노트 (2026.06.04)

이번 마이너 업데이트는 단일 모니터 및 창모드 전체화면 유저분들의 편리한 오버레이 관리를 돕는 **macOS 스타일 하단 Dock 런처** 기능이 신규 추가되었으며, **계수 계산기 화면의 레이아웃 개편** 및 **설정 화면들의 비동기 데이터 바인딩 레이스 컨디션 버그 해결** 등 사용성 향상을 위한 주요 보완 사항들이 대거 적용되었습니다.

---

## 🌟 Added (새로운 기능 추가)

- **🖥️ macOS 스타일의 하단 Dock 런처 구현**:
  - 단축키 `Ctrl + Shift + D`(기본값) 및 IPC 통신을 활용하여 하단 오버레이 바 형태로 여닫을 수 있는 런처 패널(Dock)을 구현했습니다.
  - 배치 위치 옵션(`sidebarPosition`)에 `'dock'` 모드가 정식 추가되었습니다.
  - 마우스 호버 시 아이콘 고유의 테마 색상(Theme Color)에 대응하는 배경 물듦(Glow), 하단 그림자 네온 발광, 테두리 활성화 및 활성 점(`.indicator-dot`) 표시 효과를 통해 미려한 프리미엄 UX를 완성했습니다.
  - **3구역 정렬 및 가변 레이아웃**: 독바 내 일반 기능, 퀵슬롯 바로가기 사이트, 시스템 제어 영역의 3단 구역 구조를 지니며, 등록된 퀵슬롯이 없을 때는 해당 전용 줄이 화면에서 자동으로 생략되는 가변 레이아웃이 적용되었습니다.
  - **'독 숨기기' 단축 버튼**: 독바 가장 왼쪽(첫 번째 칸)에 `chevrons-down` 버튼을 고정 노출하여 마우스 클릭만으로도 편리하게 독바 창을 닫을 수 있습니다.

- **⚙️ 환경설정 화면 내 독바 전용 설정 제어 지원**:
  - '사이드바 배치 설정' 탭에 **'하단 독(Dock) 배치'** 라디오 버튼 옵션이 신설되었습니다.
  - 단축키 설정 탭에 **'하단 독(Dock) 표시 토글'** 단축키 사용자 지정 녹화 및 초기화 UI를 제공합니다.
  - **가이드 실시간 갱신**: 사용자가 독바 단축키를 임의 변경할 때, 일반 설정의 단축키 가이드 문구(Ctrl + Shift + D 부분)도 즉시 동기화되어 실시간 변경되도록 `updateDockShortcutGuide()` 동적 로직을 추가했습니다.

## ⚡ Changed (변경 및 개선 사항)

- **🚀 독바 성능 최적화 및 레이아웃 안정성 강화**:
  - 독바 닫기(숨김) 시 단순히 창을 투명하게 가리는 대신 `close()` API를 호출해 메모리에서 인스턴스를 즉시 파괴(Destroy)하도록 조절하여 리소스 누수를 원천 차단했습니다.
  - 독바 윈도우의 Z-Order 계층을 최하단(사이드바와 동일 수준)으로 분류하여, 다른 일반 오버레이 기능 창들을 열었을 때 독바 위로 자연스럽게 얹히도록 레이어 높낮이를 보정했습니다.
  - 마우스 호버 시 박스 테두리 생성으로 인해 레이아웃이 미세하게 요동치던 현상(Layout Shift)을 보더 경계선 사전 확보 및 박스사이징 고정을 통해 해결했습니다.
  - 독바의 불투명도를 `rgba(15, 18, 30, 0.85)`로 강화하여, 화려한 게임 이펙트 위에서도 독 패널이 뚜렷하게 구별되도록 가시 대비를 높였습니다.

- **🧮 계수 계산기(Coefficient Calculator) 레이아웃 개편**:
  - 기존 세로 스크롤 위주의 콘텐츠 가이드 배치 구조를 시원하고 넓은 **5열 가로 격자 구조(`grid-cols-5`)**로 개편하여 시각적 답답함을 해결하고 여백을 최적화했습니다.
  - 하단부의 긴 정보 출처(참고자료) 표시 영역을 한눈에 들어오는 컴팩트한 인라인 가로 배치로 리디자인했습니다.
  - 계수 공식이 일부 잘려 보이던 현상을 해결하고 마우스 호버 시 툴팁으로 전체 공식을 뚜렷하게 확인할 수 있도록 HTML `title` 속성을 보강했습니다.

## 🐛 Fixed (버그 수정)

- **⏱️ 주요 설정 화면 초기 데이터 유실 및 비동기 레이스 컨디션 수정**:
  - **`boss-settings.html` (필드보스 설정) 및 `word-alarm.html` (키워드 단어 알림 설정)**:
    - 메인 프로세스로부터 설정 정보 수신을 처리하는 리스너(`onConfigData`) 등록부를 스크립트 최상단 동기 실행 영역에 배치했습니다. 이를 통해 화면 로드 완료 시 전송되는 설정 초기 데이터가 소실되어 UI가 누락되거나 체크박스가 동기화되지 않던 현상을 해결했습니다.
  - **`buff-timer.html` (버프 타이머 설정)**:
    - 비동기로 작동하는 사운드 파일 리스트(`loadSoundList`) 로딩을 `window.onload` 시점에 1회만 호출하도록 제어했습니다.
    - 엘리먼트 유효성(Null) 검증 코드를 대폭 보강하여, 사운드 로딩 지연 등으로 타이머 설정 창 내 사운드 선택 값이 UI 상에서 빈값으로 초기화되거나 덮어써지던 레이스 컨디션 오류를 완벽히 수정했습니다.
