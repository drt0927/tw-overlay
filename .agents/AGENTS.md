# TW-Overlay 에이전트 개발 수칙

이 프로젝트에서 작업을 수행하는 모든 에이전트는 다음 수칙을 무조건적으로 준수해야 합니다.

## 🎨 디자인 표준 및 가독성 최우선 준수
1. 모든 HTML, CSS, JS/TS 개발 및 UI 수정을 동반하는 개발 시, 반드시 [.gemini/DESIGN_TOKENS.md](file:///d:/github/tw-overlay/.gemini/DESIGN_TOKENS.md)에 기술된 표준 규격(시맨틱 테마 색상, Glassmorphism 토큰, 그림자 등)을 사전에 필독하고 100% 반영하여 개발해야 합니다.
2. **창 크기 및 텍스트 시인성(가독성) 규격**:
   - 사용자가 화면 내용을 쉽게 인식할 수 있도록 최소 폰트 크기는 **`0.75rem` (12px) 이상**을 반드시 유지하고, 중요 가이드 및 본문 설명은 **`0.875rem` (14px) 이상**을 권장합니다.
   - 메인 설정 창 등 복잡한 UI 창은 최소 **`800px x 600px`**, 간이 팝업 및 위젯은 최소 **`400px x 300px`** 이상의 넉넉한 최소 창 너비/높이를 보장하도록 설계하십시오. 글자가 짤리거나 화면이 조밀해서 답답해지는 현상을 절대 금지합니다.
   - 어두운 유리 테마 위에서 텍스트 색상은 충분한 대비를 가져야 합니다 (`#ffffff` 또는 `#cbd5e1` 등을 사용하여 어두운 계열의 텍스트 배치 지양).
3. 임의의 Ad-hoc 색상 스타일, 규격에 어긋나는 불완전한 여백, 가독성이 확보되지 않는 좁은 레이아웃 배치를 엄격히 금지합니다.
