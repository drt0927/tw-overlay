# TW-Overlay v1.19.1 릴리즈 노트 (2026.06.26)

이번 업데이트는 **사냥터 동선 시뮬레이션 기능의 신규 사냥터 추가** 및 **맵 이미지 저장 폴더명 명칭 개선**과 **사용자 데이터베이스 자동 안전 마이그레이션 적용**이 완료되었습니다.

## 🛠️ 주요 변경 사항 (CHANGELOG)

### Added (신규 기능)
* **사냥터 동선 시뮬레이션 신규 사냥터 추가**:
  * 기존 '시오칸하임 대장간' 외에 신규 사냥터인 **'골고다의 협곡'** 및 **'공허의 영역'** 동선 시뮬레이션 탭을 추가 통합했습니다.
  * 각각의 사냥터에 맞는 전용 맵 이미지 파일들이 지원되며, 탭 이동 시 각 사냥터 맵 상에 독립적인 동선 정보가 유지 및 복원됩니다.

### Changed (개선 및 변경)
* **맵 이미지 저장 폴더명 명칭 직관화**:
  * 맵 이미지가 저장되는 폴더 경로가 `field-macro`로 명시되어 매크로와 오해가 생길 수 있는 지점을 해소하기 위해 **`field-map`**으로 경로명을 전면 개선하고 내부 코드의 자원 경로들도 일체 업데이트했습니다.
  * 기존의 모호한 `맵.png` 파일명을 직관적으로 알아볼 수 있도록 **`대장간.png`**으로 명칭을 일원화했습니다.
  * 사냥터 탭 표시 이름을 더 직관적으로 한눈에 알아볼 수 있도록 띄어쓰기 및 구체적인 지명 명칭을 적용했습니다. (예: 대장간 -> `시오칸하임 대장간`, 골고다의협곡 -> `골고다의 협곡`, 공허의영역 -> `공허의 영역`)

### Fixed (버그 수정 / 안정화)
* **기존 사용자 데이터베이스 자동 안전 마이그레이션 적용**:
  * 기존 버전 사용자가 업데이트할 때에도 에러나 대장간 맵 이미지 엑스박스 등 누락이 없도록, 기존 로컬 DB(`diary.db`)에 설정된 예전 경로 및 탭 명칭을 앱 실행 시 자동으로 최신 상태로 강제 갱신해 주는 마이그레이션 안전 로직을 이식했습니다.
  * 기존에 사용자가 작성해둔 사냥터 동선 데이터는 고유 ID를 통해 유실 없이 그대로 보존 연동됩니다.

---

## 📥 설치 방법

1. 본 릴리즈 페이지 최하단의 **Assets** 섹션에서 `twOverlay-Setup-1.19.1.exe` 파일을 클릭하여 다운로드합니다.
2. 기존에 실행 중이던 `tw-overlay` 앱이 있다면 반드시 완전히 종료한 뒤 설치 파일을 실행합니다.
3. 화면의 안내에 따라 설치를 완료하면 새로운 기능과 패치가 적용됩니다.
