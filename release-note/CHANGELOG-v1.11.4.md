# 📦 TW-Overlay Release Note [v1.11.4]

v1.11.4 패치는 멀티 모니터 및 고해상도(DPI) 환경에서 발생하던 사이드바 위치 이탈 문제를 근본적으로 해결하고, 해상도가 낮은 환경에서도 UI가 잘리지 않도록 **지능형 창 크기 조절**과 **사이드바 스크롤 시스템**을 새롭게 도입했습니다.

## 🚀 Added (신규 기능)

- **사이드바 지능형 스크롤 시스템**: 
  - **슬림 스크롤바**: 저해상도 환경이나 게임 창 크기 축소 시 아이콘이 잘리지 않도록 중간 메뉴 영역에 스크롤 기능을 추가했습니다.
  - **시각적 피드백**: 스크롤 가능 여부를 직관적으로 인지할 수 있도록 사이드바 우측에 매우 얇은 보라색 스크롤바를 배치했습니다. (평소에는 투명하며 마우스 호버 시 선명하게 노출)
  - **레이아웃 고정**: 스크롤 시에도 상단 접기 버튼과 하단 시스템 버튼(설정, 종료, 후원)은 항상 제자리에 고정되어 즉각적인 접근이 가능합니다.

- **UI 창 높이 자동 최적화 (Auto-Clamping)**: 
  - **해상도 감지**: 숙제 체크리스트나 설정 창 등 높이가 긴 창을 열 때, 현재 모니터의 작업 영역(Work Area) 높이를 실시간으로 감지합니다.
  - **자동 축소**: 설정된 창 높이가 모니터 해상도를 초과할 경우, 화면을 벗어나지 않도록 모니터 높이에 맞춰 창 크기를 자동으로 최적화하여 생성합니다.

- **모험 일지 월간 수익 추이 그래프**:
  - **데이터 시각화**: 한 달간의 일별 수익 데이터를 선 그래프(Line Chart)로 시각화하여 사냥 리듬과 수익 집중도를 한눈에 파악할 수 있습니다.
  - **지능형 단위 포맷팅**: Y축 및 툴팁에 '만/억/조' 단위를 적용하여 큰 수치도 직관적으로 읽을 수 있도록 개선했습니다.

## 🎨 Changed (변경 및 개선)

- **알림 토스트(Toast) 레이아웃 개선**:
  - 사이드바를 왼쪽에 배치했을 때 보스 등장 알림 등 토스트 메시지가 화면 밖으로 나가거나 사이드바와 겹치던 위치 오프셋을 전면 수정했습니다.
  - 이제 사이드바 위치(좌/우)에 따라 알림 창이 가장 자연스러운 위치에 정렬됩니다.
- **창 생성 위치 알고리즘 강화**: 멀티 모니터 환경에서 새 창을 띄울 때, 게임 창이 위치한 모니터 밖으로 창이 나가지 않도록 경계값 검증(Clamping) 로직을 강화했습니다.
- **DB 스키마 확장 및 통계 성능 최적화**:
  - **수치 전용 컬럼 도입**: `activity_logs` 테이블에 `amount` 컬럼을 추가하여 텍스트 파싱 없이 SQL 수준에서 즉시 통계를 산출하도록 개선했습니다. (기존 데이터 자동 마이그레이션 포함)
  - **자동 수익 기록 축약 UI**: 모험 일지의 수익 내역과 활동 타임라인에서 연속된 `[자동]` 기록을 하나로 묶어 보여주는 그룹화 로직을 도입했습니다.
  - **아코디언(접기/펴기) 기능**: 요약된 자동 기록을 클릭하여 상세 내역을 볼 수 있는 기능을 추가하여 화면 공간 효율을 대폭 높였습니다.

## 🐞 Fixed (버그 수정)

- **멀티 모니터 좌표 침범 버그 수정**: 
  - 보조 모니터나 높은 DPI 스케일링(150% 이상) 환경에서 사이드바가 게임 창 안쪽으로 침범하거나 크기가 축소되던 치명적인 좌표 계산 오류를 해결했습니다.
  - 수동 픽셀 계산 방식에서 Electron 내장 정밀 변환 API(`screenToDipRect`)로 전환하여 픽셀 단위의 정확한 동기화를 보장합니다.
- **DPI 스케일링 혼선 수정**: 멀티 모니터 간 이동 시 윈도우 스케일 인수가 잘못 적용되어 오버레이 위치가 튀는 현상을 수정했습니다.
- **사이드바 리사이즈 깜박임 개선**: 사이드바 확장/축소 시 좌표 재계산 과정에서 발생하던 시각적 끊김 현상을 최적화했습니다.
- **경험치 기록 타이밍 오차 수정**: 경험치 획득량 계산 전 분(minute) 전환을 먼저 체크하도록 수정하여, 매 분의 첫 번째 경험치가 누락되거나 이전 분에 포함되던 데이터 정확도 문제를 해결했습니다.
- **로그 디코딩 엔진 표준화**: 게임 로그를 읽어올 때 불필요한 바이너리 변환 과정을 제거하고 Buffer를 직접 디코딩하는 방식으로 최적화하여 로그 스트리밍의 안정성을 높였습니다.
- **차트 렌더링 안정성 강화**: 통계 탭이 활성화되지 않은 상태에서 차트를 그릴 때 발생할 수 있는 DOM 접근 오류(null pointer)를 방지하는 방어 코드를 추가했습니다.
