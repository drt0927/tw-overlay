# TW-Overlay v1.18.13 릴리즈 노트 (2026.06.25)

이번 업데이트는 **모험일지에 기록되는 경험의 정수 획득 경로 다양화(직접/자동 교환 및 퀘스트/아이템 보상)와 사냥 HUD 세션 통계 분리**가 적용되었습니다.

## 🛠️ 주요 변경 사항 (CHANGELOG)

### Added (새 기능)
* **경험의 정수 다양한 획득 경로 감지 지원**:
  * 기존 경험치 감소 방식(100억 XP 교환) 외에도 보급품 탈환 성공 등 퀘스트 보상이나 일반 아이템 드랍으로 획득하는 경험의 정수 획득 채팅 로그를 감지하기 위해 새로운 `ESSENCE_GAINED` 내부 이벤트를 추가했습니다.
  * 다음과 같이 어미 형태가 다른 다양한 형태의 획득 문구를 정확히 분석하여 감지합니다:
    - `"보급품 탈환 성공 보상으로 경험의 정수 1개와 3000만 Seed를 획득했습니다."`
    - `"보급품 탈환 성공 보상으로 경험의 정수 1개를 획득했습니다."`
    - `"[경험의 정수] 아이템을 획득하였습니다."`
    - `"[경험의 정수] 아이템을 추가로 획득하였습니다."`
    - `"[경험의 정수] 아이템을 1개 획득하였습니다."`
    - `"[경험의 정수] 아이템을 10개 획득하였습니다."`

### Changed (개선 및 변경점)
* **경험치 HUD 사냥 통계와 모험일지 기록 분리**:
  * **사냥 직접/자동 교환(1, 2번 유형)**: 경험치 100억이 감소되는 시점에 감지되어 **경험치 HUD의 세션 카운트**에 누적되고 **모험일지 타임라인**에도 기록됩니다.
  * **퀘스트/아이템 보상(3번 유형)**: 사냥으로 인한 경험치 교환이 아니므로 사냥 통계(정수 예측기 등)에 영향을 주지 않도록 **HUD 통계 카운트에서 배제**하고, 오직 **모험일지 타임라인**에만 득템 기록으로 분리 저장됩니다.
* **수량 파싱 로직 정교화**:
  * 획득 로그 중 수량이 명시된 문구(`경험의 정수 N개`, `아이템을 N개`)에서는 정확한 숫자를 추출하며, 수량이 생략된 경우(`추가로 획득하였습니다` 등)에는 기본값 `1개`로 매핑하여 모험일지에 수량을 정확히 적재합니다.

### Fixed (버그 수정)
* **경험의 정수 자동 교환 버프 시 중복 기록 버그 수정**:
  * 사냥 중 자동 교환 버프가 동작할 때 발생하는 중복 로그(`경험치 100억이 차감되고...`)가 이중으로 기록되지 않도록 정밀 필터를 설계하여 중복 등록 현상을 근본적으로 해결했습니다.

## 📥 설치 방법

1. 본 릴리즈 페이지 최하단의 **Assets** 섹션에서 `twOverlay-Setup-1.18.13.exe` 파일을 클릭하여 다운로드합니다.
2. 기존에 실행 중이던 `tw-overlay` 앱이 있다면 반드시 완전히 종료한 뒤 설치 파일을 실행합니다.
3. 화면의 안내에 따라 설치를 완료하면 새로운 기능과 패치가 적용됩니다.
