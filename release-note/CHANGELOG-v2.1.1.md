# 📦 TW-Overlay Release Note - v2.1.1 (2026-07-09)

이번 릴리즈는 사냥터 동선 시뮬레이션의 마우스 투과 오버레이 모드를 고도화하고, 윈도우 파괴 및 라이프사이클 관련 예외 처리를 보강하여 데스크톱 실행 안전성을 향상시킨 패치 업데이트입니다.

---

## 🚀 주요 업데이트 내역

### 🗺️ Added (추가 기능)
- **사냥터 동선 시뮬레이션 오버레이 드래그 위치 이동 기능**
  - 오버레이 모드 활성화 상태에서 마우스를 가져가 활성화된 미니 제어 패널 상단의 그랩 아이콘(`grip-horizontal`)을 잡고 드래그하면, 오버레이 창 전체의 화면상 위치를 자유롭게 이동시킬 수 있는 윈도우 드래그 이벤트를 신설했습니다.

### 🇰🇷 Changed (변경 및 정비)
- **일반 편집 모드 1줄 옵션바 복원 및 기본 창 규격 원복**
  - 일반 편집 모드에서는 맵 크기(스케일) 및 지도 투명도 조절 슬라이더를 제거하여, 맵을 100% 원본 크기로 넓고 쾌적하게 편집할 수 있도록 간소화했습니다.
  - 옵션바를 다시 깔끔한 가로 1줄 레이아웃으로 복원하고, 창의 기본 크기도 원래의 규격인 **가로 `860px`, 세로 `800px`로 복원**했습니다.
  - '동선 초기화' 버튼을 하단 단축키 가이드 배너의 우측 끝에 조화롭게 다시 이전하여 옵션바의 심플함을 강화했습니다.
- **오버레이 모드 전용 스케일/투명도 제어 및 편집 복귀 시 강제 원복**
  - 맵 축소(스케일) 및 지도 투명도 제어 슬라이더는 오버레이 모드 내 미니 패널에서만 나타나도록 한정 분리했습니다.
  - 오버레이에서 변경한 크기/투명도는 전용 로컬 캐시(`hps-overlay-scale`, `hps-overlay-opacity`)에 별도 격리 저장되어, 오버레이 모드를 다시 켤 때 사용자가 맞춘 최적화된 오버레이 비율로 자동 축소 적용됩니다.
  - 미니 패널의 `편집 복귀` 버튼을 누르면 즉시 지도의 스케일을 무조건 `1.0` (100%), 투명도를 `0.8`로 초기화하며 창의 크기도 무조건 원래 기본 규격인 `860x800`으로 깨끗하게 롤백합니다.
- **스케일 조절 1% 단위 정밀화 및 축소 하한선 20% 확장**
  - 오버레이 모드 내 맵 축소 조절 한도를 기존 40%에서 **20%**까지 대폭 완화하여 한정된 게임 화면을 더욱 유연하게 활용할 수 있게 하였습니다.
  - 축소 정밀도를 10% 단위에서 **1%** 단위로 세밀화하여 원하는 픽셀 크기에 유연하게 매핑되도록 step 간격을 고도화했습니다.
- **맵 축소 비율에 따른 선 두께 및 화살표 머리 크기 동조 감축**
  - 지도가 축소될 때 선 두께(`stroke-width`)와 화살표 마커의 크기(`markerWidth`, `markerHeight`, `refX`, `refY`)가 동적으로 비례 축소되도록 자동 보간식을 연동했습니다. (선 굵기 기본 배율도 3.5에서 5.0으로 상향하여 가시성 강화)

### 🐛 Fixed (버그 수정)
- **오버레이 모드 해제 시 편집 창 포커스 상실(게임창 뒤로 숨음) 수정**
  - 오버레이 항상 위에 고정(`setAlwaysOnTop(false)`)이 해제되는 순간, 시뮬레이터 창이 포커스를 잃고 게임 화면 뒤로 가려 숨는 문제를 해결하기 위해 해제 즉시 메인 프로세스에서 수동 활성화(`win.show()`, `win.focus()`)하여 전면으로 컴백하도록 강제 포커싱을 추가했습니다.
- **오버레이 모드 시 노드 및 노드-히트박스 노출 결함 해결**
  - 인게임 투과 모드 시 노드의 번호 텍스트뿐만 아니라 노드 점 원형(`.node`)과 마우스 감지 히트박스(`.node-hitbox`)까지 통째로 숨겨, 오직 선명한 경로 라인과 화살표만 깔끔히 노출되도록 CSS 스타일을 수정했습니다.
  - 노드가 가려진 오버레이 화면에서도 화살표 머리가 선 끝머리에 정확하게 바싹 밀착되어 도킹되도록 조건부 정렬 오프셋(`refX = 10`)을 적용하여 둥둥 떠 보이는 그래픽 이격을 해결했습니다.
- **프로그램 초기 구동 시 단축키 등록 지연 버그 수정**
  - 처음 프로그램을 실행하고 인게임 창이 포커스된 상태에서 전역 단축키를 눌렀을 때, 콘텐츠 윈도우 생성 지연으로 인해 조작 반응이 미비하던 현상을 개선하고 단축키 등록 시점을 즉시 보장하도록 패치했습니다.
- **Win32 윈도우 Object destroyed 예외 처리 보강**
  - IPC 핸들러 및 윈도우 관리 레이어에서 백그라운드 파괴 처리된 BrowserWindow 객체를 참조하여 발생하는 `Object has been destroyed` 런타임 크래시 예외를 완전 방지하기 위해 `isDestroyed()` 유효성 검사를 일괄 보강했습니다.
