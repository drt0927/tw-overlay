# 📦 TW-Overlay 릴리즈 노트 - v1.18.14 (2026-06-26)

## ⏱️ Added (추가된 기능)
* **사미선령지채 버프 타이머 연동**
  * 요리로 제작 및 섭취하는 '사미선령지채' (지속시간 5분, 모든 능력치 +30) 아이템의 실시간 채팅 로그 사용 감지(본인/파티원) 및 타이머 구동 기능을 추가했습니다.
  * 버프 타이머 설정 메뉴에서 사미선령지채를 다른 버프들과 마찬가지로 토글하여 개별적으로 보이기/숨기기 관리할 수 있습니다.
  * 미리보기(테스트) 모드 구동 시 사미선령지채 버프 또한 함께 시뮬레이션되도록 추가했습니다.

## ⚙️ Changed (변경된 기능 및 리팩토링)
* **버프 효과 데이터 구조화 (`effects` 필드 도입)**
  * `buffs.json`에 정의된 기존의 텍스트 기반 효과(`effect`)와 별개로, 각 효과의 숫자 데이터를 나타내는 구조화된 `effects` 프로퍼티를 추가했습니다.
  * `exp` (경험치), `rare` (레어), `stat` / `statRate` (능력치 고정/비율), `damage` (대미지), `critical` (크리티컬), `speed` (이속), `attribute` (속성) 등으로 쪼개어 데이터를 구성함으로써 연산 신뢰도를 확보했습니다.
* **버프 명칭 일관성 수정**
  * `buffs.json` 내 기존 `"능력치 +N"`으로 표현된 설명을 유저들에게 직관적인 `"모든 능력치 +N"`으로 표기명을 통일시켰습니다.

## 🛡️ Fixed (버그 수정)
* **버프 계수 계산기 정규식 파싱 오류 및 스탯 중복 가산 버그 수정**
  * 기존 계산기에서 정규식으로 버프의 `effect` 스트링을 파싱하여 스탯 보너스를 더해오던 로직을, `effects.stat` 및 `effects.statRate` 프로퍼티의 값을 직접 읽어 연산하도록 전면 리팩토링했습니다.
  * 기존 정규식이 퍼센트 단위 버프(예: `능력치 +10%`) 파싱 시, 숫자 뒷자리에 붙은 `%`를 무시하고 앞자리 숫자(`1`)를 고정 수치 가산(`+1`)으로 잘못 감지하여 스탯을 이중 합산하던 잠재적 버그가 이번 데이터 구조화 리팩토링을 통해 원천 해결되었습니다.
