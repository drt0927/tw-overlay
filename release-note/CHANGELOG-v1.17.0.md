# TW-Overlay v1.17.0 릴리즈 노트 (2026.06.04)

이번 마이너 업데이트는 인게임에서 투명하고 편리하게 대화 로그를 조회할 수 있는 **인게임 채팅 오버레이 위젯**이 신규 추가되었으며, **사기꾼 탐지 AI 기능의 테스트/안전성 강화**, **에타(ETA) 레벨 배지 지원**, 그리고 대용량 로그 스캔 시 렉 현상을 원천 방지하는 **지연 파싱(Lazy Parsing) 성능 고속화** 등 채팅 관련 주요 편의성과 견고함이 대거 강화되었습니다.

---

## 🌟 Added (새로운 기능 추가)

- **💬 인게임 채팅 오버레이 위젯 구현**:
  - 게임 화면 위에 얹히는 별도의 투명 채팅 오버레이 창(`chatOverlay`, `chatOverlaySub`, `chatOverlaySub2` 3종의 서브 창 지원)이 추가되었습니다.
  - **다중 탭 분류 기능**: 전체(Basic)/일반(General)/귓속말(Whisper)/팀(Team)/클럽(Club)/외치기(Shout)/시스템(System) 탭을 지원하여 원하는 대화 카테고리만 손쉽게 모아볼 수 있습니다.
  - **마우스 투과 모드 토글 연동**: 단축키 `Ctrl + Shift + T`(기본값)를 통해 채팅 오버레이 창도 마우스 클릭을 투과할 수 있게 연동되어, 플레이 도중 시야 방해 없이 투명 가이드를 제공합니다.
  - **무한 스크롤 지원**: 대화창 상단 스크롤(위로 올리기) 시, 오늘 쌓인 이전 대화 로그를 150개씩 부드럽게 불러와 렌더링하며 화면 튕김 방지 스크롤 고정 처리가 내장되어 있습니다.

- **🛡️ 에타(ETA) 레벨 배지 표시 및 색인 캐시 매니저**:
  - 일반 대화 및 귓속말 수신 시, 대화 상대방의 실시간 에타 레벨 배지를 닉네임 옆에 함께 렌더링하여 거래 신원 검증용 비주얼 장치를 지원합니다.
  - `etaCacheManager`를 신설하여 원격 저장소(`drt0927/tw-overlay-data`)의 실시간 에타 랭킹 JSON 데이터를 비동기로 패치하고, 메모리에 `O(1)` 구조로 색인화하여 대화 발생 시의 성능 오버헤드를 제로화했습니다.
  - **트래픽 및 기동 최적화**: 로컬 캐시 파일의 수집일이 오늘인 경우 불필요한 원격 네트워크 조회를 생략합니다.

---

## ⚡ Changed (변경 및 개선 사항)

- **⚡ 대량 로그 스캔 렉 현상 해결 (Lazy Parsing 최적화)**:
  - 앱 기동 시 오늘 누적된 로그 파일(수만 줄)을 역순 재연할 때 발생하던 CPU 연산 병목을 해결하기 위해 **지연 파싱 (Lazy Parsing)** 방식을 도입했습니다.
  - `includes()` 단일 단순 매칭으로 카테고리를 초고속 대분류한 뒤, 해당 카테고리가 이미 수집 한도(150개)를 다 채웠다면 시간 정규식이나 HTML 파싱 등 무거운 문자열 연산을 아예 생략하고 즉시 다음 줄로 이동(Early Escape)합니다. 이로 인해 스캔 소요 시간이 수 초에서 0.00초대로 비약적으로 단축되었습니다.

- **🤖 사기꾼 탐지 기능의 UI 직관성 개선**:
  - 설정 화면의 토글 명칭을 유저가 더 이해하기 쉽도록 직관적(긍정형)으로 개선했습니다:
    - "1:1 대화 실시간 탐지" 및 "AI 대화 위험도 분석" (상세 가이드 캡션 추가).
  - 내부 설정 키 `scamLlmDisabled` 논리 매핑 값을 뒤집어 직관성을 통일했습니다.

## 🐛 Fixed (버그 수정)

- **🧠 사기꾼 탐지 테스트 세션 가상 분석 예외 안전화**:
  - AI 기능 비활성화 또는 모델 미설치 상태에서도 사기꾼 탐지 레이아웃과 배지를 검증할 수 있도록 테스트 세션 주입 시 가상 분석(Mock Analysis) 시나리오를 구동하게 조절했습니다.
  - 가상 분석 중 네트워크나 파일 에러 등이 발생해도 UI 및 내부 분석 락 상태가 동결되지 않도록 `try-catch-finally` 안전 구조로 락 해제 메커니즘을 확립했습니다.

- **🎨 획득 로그의 색상 표시 일치화**:
  - 실시간 로그 파서와 과거 로그 스캐너 간의 획득 관련 문구(아이템: 황금색, 시드: 회색) 색상 표시가 서로 다르게 렌더링되던 불일치 버그를 수정하여 UI 일관성을 보장했습니다.

- **🔗 에타 캐시 원격 브랜치 404 리스크 차단**:
  - 에타 랭킹 데이터를 다운로드하는 Git 원격 리포지토리 브랜치 주소를 피처 브랜치에서 공식 안정 브랜치인 `main`으로 고정하여, 피처 브랜치 삭제 시 다운로드가 실패하는 리스크를 방어했습니다.
